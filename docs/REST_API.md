# OpenPaste REST API

## Overview

OpenPaste exposes a REST API for external integration and automation. The API runs on `localhost:7890` by default and provides endpoints for clipboard operations, search, storage, and system management.

## API Specification

### Base URL

```
http://localhost:7890/api/v1
```

### Authentication

**Header:**
```
Authorization: Bearer <token>
```

**Token Generation:**
- Generated by daemon
- Stored in settings
- Can be revoked
- Optional for localhost (configurable)

### Response Format

**Success Response:**
```json
{
  "success": true,
  "data": { ... }
}
```

**Error Response:**
```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "Error message",
    "details": { ... }
  }
}
```

### Items

#### GET /items

List clipboard items.

**Request:**
```http
GET /api/v1/items?limit=50&offset=0&content_type=text
Authorization: Bearer <token>
```

**Query Parameters:**
- `limit` (optional): Number of items to return (default: 50, max: 100)
- `offset` (optional): Offset for pagination (default: 0)
- `content_type` (optional): Filter by content type (text, image, html, file)
- `pinned` (optional): Filter by pinned status (true/false)
- `favorite` (optional): Filter by favorite status (true/false)
- `collection_id` (optional): Filter by collection ID
- `tag` (optional): Filter by tag name
- `after` (optional): Filter by date (Unix timestamp in ms)
- `before` (optional): Filter by date (Unix timestamp in ms)

**Response:**
```json
{
  "success": true,
  "data": {
    "items": [
      {
        "id": 123,
        "content_type": "text",
        "content_preview": "Hello, World!",
        "created_at": 1704067200000,
        "source_app": "Chrome",
        "pinned": false,
        "favorite": false,
        "tags": ["important"]
      }
    ],
    "total": 100,
    "limit": 50,
    "offset": 0
  }
}
```

## WebSocket

### Connection

**Endpoint:**
```
ws://localhost:7890/api/v1/ws
```

**Authentication:**
- Query parameter: `?token=<token>`
- Or header: `Authorization: Bearer <token>`

### Events

**Message Format:**
```json
{
  "event": "clipboard.added",
  "data": { ... },
  "timestamp": 1704067200000
}
```

**Event Types:**
- `clipboard.added`
- `clipboard.changed`
- `item.pinned`
- `item.deleted`
- `encryption.locked`
- `encryption.unlocked`
- `sync.started`
- `sync.completed`

### Example

**Client Connection:**
```javascript
const ws = new WebSocket('ws://localhost:7890/api/v1/ws?token=abc123');

ws.onmessage = (event) => {
  const message = JSON.parse(event.data);
  console.log(message.event, message.data);
};
```

## Error Codes

| Code | Name | Description |
|------|------|-------------|
| 400 | Bad Request | Invalid request |
| 401 | Unauthorized | Invalid or missing token |
| 403 | Forbidden | Insufficient permissions |
| 404 | Not Found | Resource not found |
| 409 | Conflict | Conflict with existing data |
| 422 | Unprocessable Entity | Invalid data |
| 429 | Too Many Requests | Rate limit exceeded |
| 500 | Internal Server Error | Server error |
| 503 | Service Unavailable | Service temporarily unavailable |

## Rate Limiting

**Default Limits:**
- 100 requests/second per token
- 1000 requests/minute per token

**Headers:**
```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1704067200
```

**Rate Limit Exceeded:**
```json
{
  "success": false,
  "error": {
    "code": "RATE_LIMIT_EXCEEDED",
    "message": "Rate limit exceeded",
    "details": {
      "limit": 100,
      "remaining": 0,
      "reset": 1704067200
    }
  }
}
```

## Filtering

**Content Type Filter:**
```http
GET /api/v1/items?content_type=text
```

**Multiple Content Types:**
```http
GET /api/v1/items?content_type=text,image
```

**Date Range Filter:**
```http
GET /api/v1/items?after=1704067200000&before=1704153600000
```

**Tag Filter:**
```http
GET /api/v1/items?tag=important
```

**Multiple Tags:**
```http
GET /api/v1/items?tag=important&tag=work
```

**Collection Filter:**
```http
GET /api/v1/items?collection_id=1
```

## Sorting

**Sort Parameter:**
```http
GET /api/v1/items?sort=created_at&order=desc
```

**Sort Options:**
- `created_at` (default)
- `last_accessed_at`
- `access_count`
- `size_bytes`

**Order Options:**
- `asc` (ascending)
- `desc` (descending, default)

## Examples

### cURL Examples

**Get clipboard:**
```bash
curl -H "Authorization: Bearer <token>" \
  http://localhost:7890/api/v1/clipboard
```

**Search items:**
```bash
curl -X POST \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"query": "hello"}' \
  http://localhost:7890/api/v1/search
```

**Create item:**
```bash
curl -X POST \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"content": "Hello, World!", "content_type": "text"}' \
  http://localhost:7890/api/v1/items
```

### Python Examples

```python
import requests

API_URL = "http://localhost:7890/api/v1"
TOKEN = "your_token"

headers = {
    "Authorization": f"Bearer {TOKEN}",
    "Content-Type": "application/json"
}

# Get clipboard
response = requests.get(f"{API_URL}/clipboard", headers=headers)
print(response.json())

# Search
response = requests.post(
    f"{API_URL}/search",
    headers=headers,
    json={"query": "hello"}
)
print(response.json())
```

### JavaScript Examples

```javascript
const API_URL = "http://localhost:7890/api/v1";
const TOKEN = "your_token";

const headers = {
  "Authorization": `Bearer ${TOKEN}`,
  "Content-Type": "application/json"
};

// Get clipboard
fetch(`${API_URL}/clipboard`, { headers })
  .then(r => r.json())
  .then(data => console.log(data));

// Search
fetch(`${API_URL}/search`, {
  method: "POST",
  headers,
  body: JSON.stringify({ query: "hello" })
})
  .then(r => r.json())
  .then(data => console.log(data));
```

## Security

```json
{
  "api": {
    "https_enabled": true,
    "cert_file": "/path/to/cert.pem",
    "key_file": "/path/to/key.pem"
  }

```

**Default:** CORS disabled (localhost only)

**Enable CORS:**
```json
{
  "api": {
    "cors_enabled": true,
    "cors_origins": ["http://localhost:3000"]
  }
}
```
